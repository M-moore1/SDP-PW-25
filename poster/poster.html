Secure Embedded Wireless Vehicle Control

CSE Team 20: Krish, Mason, Sai, Drew  |  Advisors: Amir Herzeberg
University of Connecticut · Department of Computer Science & Engineering
📋
Background

Our goal was to create a remote-controlled vehicle that demonstrates real-world security features in embedded wireless systems. By building on a robot platform, we aimed to show how encryption, authentication, and secure communication protocols can be applied in a practical, hands-on setting.

🎯
Goal / Motivation

Design and implement a secure wireless control system that ensures authenticated, encrypted command transmission. The system must prevent replay attacks, man-in-the-middle attacks, and unauthorized command injection.

🔧
System Architecture

The system comprises a press-and-hold WASD controller interface, a server, and a Bluetooth-connected robot.

Security Protocols
AES-256-GCM Encryption

All command payloads are encrypted using AES-GCM

BLE (Bluetooth Low Energy)

Wireless communication between server and robot uses BLE for low-power, short-range data transfer

TLS (Transport Layer Security)

TLS secures the TCP channel between client and server
